# Claude Skill: Context Engineering Review

## Purpose

Use this skill to evaluate whether the prototype demonstrates mature context engineering.

## Core Principle

Prompt engineering is not enough. Strong AI workflows require reusable context systems that define brand voice, source material, constraints, channel needs, examples, and review rules.

## Review Criteria

### Context Quality

Check whether the project explains:

- What source context is needed
- How brand rules are captured
- How approved claims are separated from assumptions
- How examples influence output
- How channel requirements affect generation
- How context is updated over time

### Workflow Clarity

Check whether the project shows:

- Brief intake
- Brand context pack
- Prompt/template layer
- Output schema
- Review checklist
- Handoff documentation

### Risk Control

Check whether the project avoids:

- Generic “make this better” prompting
- Unsupported claims
- Blind trust in AI output
- Auto-publishing
- Brand inconsistency
- Unclear source attribution

## Output Standard

The prototype should show that AI quality comes from structured context, not magic prompting.
