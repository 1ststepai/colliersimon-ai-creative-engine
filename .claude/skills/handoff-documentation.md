# Claude Skill: Handoff Documentation

## Purpose

Use this skill to evaluate whether the repo and prototype are easy for another person to understand, run, review, and extend.

## Review Checklist

### README

The README should explain:

- What the project is
- Why it exists
- Who it is for
- How it maps to the Creative Technologist role
- How to run locally
- How to deploy
- What is mocked
- What is not official
- What would be needed for production

### Code Handoff

The codebase should have:

- Clear component names
- Logical file structure
- Minimal unnecessary complexity
- Static/demo-safe behavior
- No secrets
- No private data
- No broken links
- No hidden dependencies

### Prototype Handoff

The prototype should make clear:

- Inputs
- Workflow stages
- QA points
- Outputs
- Human review requirements
- Documentation expectations

## Output Standard

A reviewer should not need the creator on a call to understand the project.
