# Claude Skill: Repo Review

## Purpose

Use this skill to review the GitHub repo as part of the candidate presentation.

## Review Areas

### First Impression

- Does the repo name make sense?
- Is the description clear?
- Does the README immediately explain the concept?
- Does the repo feel intentional?

### Professional Boundaries

Confirm the repo does not imply:

- Official Collier.Simon approval
- Commissioned client work
- Use of private data
- Use of proprietary assets
- Production deployment for a real client

### Hiring Signal

The repo should communicate:

- Technical ability
- Documentation discipline
- AI systems thinking
- Creative production awareness
- QA mindset
- Handoff maturity
- Judgment

### Cleanliness

Check for:

- No secrets
- No broken files
- No messy scratch files
- No unfinished TODOs
- No dead links
- No large unnecessary assets
- No dependency bloat

## Final Review Prompt

If reviewing this repo, answer:

“Would this make a hiring team more confident that Evan can build practical AI workflows for creative teams?”

If not, identify the exact changes needed.
