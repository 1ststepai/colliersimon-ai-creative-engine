# Project Brief: AI Creative Engine

## Project Type

Spec concept / candidate prototype.

## Purpose

Build a polished Creative Technologist portfolio prototype that demonstrates how Evan Pancis thinks about AI-enabled creative production workflows.

The prototype should show the ability to translate creative and production needs into repeatable systems involving:

- Brief intake
- Brand context
- AI-assisted variant generation
- Human QA
- Documentation
- Production handoff

## Audience

Primary audience:

- Collier.Simon hiring team
- Head of Creative Technology
- Creative leadership
- Production or innovation team reviewers

Secondary audience:

- Recruiters reviewing role alignment
- Technical reviewers reviewing repo quality
- Non-technical reviewers evaluating communication clarity

## What This Should Communicate

This project should communicate:

- I can build.
- I can document.
- I can think in workflows.
- I understand AI beyond prompting.
- I know AI requires guardrails.
- I can explain technical concepts to creative and non-technical teams.
- I know the difference between a flashy demo and a usable system.
- I understand handoff, QA, and production readiness.

## What This Should Avoid

This project should not:

- Claim to be official Collier.Simon work
- Use private client data
- Overclaim production readiness
- Suggest AI replaces creative judgment
- Use generic AI buzzwords without workflow substance
- Look like a template or starter project

## Success Criteria

The prototype is successful if a reviewer can quickly understand:

- What problem it solves
- How the workflow operates
- Where AI helps
- Where humans review
- How outputs get handed off
- Why the repo itself shows professional working habits
