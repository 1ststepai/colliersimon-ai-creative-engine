# Skill: Next.js Polish

## Purpose

Use this skill when improving the quality, structure, and presentation of the Next.js prototype.

## Standards

The prototype should feel intentional, modern, and production-aware.

## Requirements

### App Structure

- Use Next.js App Router.
- Keep components modular.
- Separate content sections into reusable components.
- Avoid one massive unmaintainable file.
- Keep static data organized.

### TypeScript

- Use clear types where helpful.
- Avoid unnecessary complexity.
- Avoid `any` unless there is a clear reason.
- Keep props readable.

### Styling

- Use Tailwind CSS.
- Maintain consistent spacing.
- Use strong typography hierarchy.
- Avoid clutter.
- Use responsive layouts.
- Keep cards aligned.
- Avoid text overflow and clipping.
- Preserve accessibility contrast.

### UI Quality

Check:

- Hero has immediate impact.
- Sections have clear rhythm.
- Cards are scannable.
- Prototype interface feels believable.
- Buttons and anchors work.
- Footer is clean.
- Mobile view is polished.

### Performance

- No unnecessary heavy dependencies.
- No external API calls.
- No paid services.
- No secrets.
- No client data.
- Static-safe deployment.

## Final QA

Before finishing, run or verify:

- npm run lint
- npm run build
- Manual responsive review
- Spelling review
- Link review
- Metadata review
