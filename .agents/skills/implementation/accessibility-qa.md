# Skill: Accessibility QA

## Purpose

Use this skill to ensure the prototype is accessible, readable, and professional.

## Core Checks

### Semantic Structure

- One clear h1.
- Logical h2/h3 hierarchy.
- Sections use meaningful structure.
- Buttons and links have clear labels.

### Contrast

- Text must be readable against background.
- Avoid low-contrast gray text.
- Accent colors should not carry meaning alone.

### Responsiveness

Review:

- Desktop
- Tablet
- Mobile
- Narrow viewport

Check for:

- Overlapping elements
- Clipped text
- Horizontal scroll
- Cards too compressed
- Buttons wrapping awkwardly

### Keyboard and Links

- Anchor links should work.
- Focus states should be visible.
- Buttons should be actual links or buttons depending on behavior.

### Motion

- Avoid excessive motion.
- Do not use motion that distracts from content.
- Respect users who may prefer reduced motion if animations are added.

## Accessibility Standard

The prototype should look premium without sacrificing clarity.

Readable beats clever.
