# Skill: Brand Context Systems

## Purpose

Use this skill when building AI workflows that need to produce brand-consistent outputs.

The goal is to prevent generic AI output by giving the system structured brand context before generation.

## Core Principle

The quality of AI output depends heavily on the quality of context. Prompting alone is not enough. A reusable context system is what makes output consistent, reviewable, and production-ready.

## Context Pack Structure

A brand context pack should include:

### Brand Foundation

- Brand name
- Category
- Audience
- Product or service truth
- Core value proposition
- Tone and voice
- Strategic positioning

### Messaging System

- Primary message
- Supporting messages
- Proof points
- Claims that are allowed
- Claims that are not allowed
- Common objections
- Differentiators

### Creative Rules

- Words to use
- Words to avoid
- Approved taglines
- Visual tone
- Formatting rules
- Accessibility requirements
- Channel-specific limits

### Examples

- Approved past copy
- Winning ads
- Landing page examples
- Emails or CRM examples
- Video scripts
- Social examples
- Negative examples

### Compliance and Risk Notes

- Required disclaimers
- Legal review requirements
- Sensitive topics
- Restricted claims
- Approval dependencies

## Workflow

1. Gather source materials.
2. Extract durable brand rules.
3. Convert rules into structured context.
4. Separate facts from preferences.
5. Add do/do-not guidance.
6. Test sample outputs.
7. Refine context based on QA.
8. Document how future users should apply the context pack.

## Good Context Engineering Looks Like

Good context engineering creates outputs that feel:

- Specific
- On-brand
- Channel-aware
- Strategically aligned
- Less generic
- Easier to QA
- Easier to repeat

## Anti-Pattern

Do not rely on a single long prompt with vague instructions like:

“Make this sound on-brand.”

Instead, create structured context that defines what “on-brand” actually means.
